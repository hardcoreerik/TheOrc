## Adding a New Tool

1. Write handler in appropriate `Tools\*.cs`:
   ```csharp
   public static Task<string> MyTool(Dictionary<string, object?> args, CancellationToken ct)
   ```

2. Register in `MainWindow.RegisterAllTools()`:
   ```csharp
   _registry.Register(new ToolDefinition {
       Name        = "my_tool",
       Description = "What it does",
       Parameters  = new() { { "param_name", "string" } },
       Required    = ["param_name"],
       Handler     = MyToolsClass.MyTool
   });
   ```

3. Add to `ToolRegistry.GetForProfile()` allowed list for `Coding` or `Full`
4. Add a hint in `ToolRegistry.BuildHint()` for any common hallucinated alias names
5. Update this file with the new tool

## WPF Patterns & Gotchas

| Situation | Right approach |
|---|---|
| Equal-width tabs | `UniformGrid Rows="1"` as ItemsPanel + disabled H-scroll on wrapper |
| Async UI approval | `TaskCompletionSource<T>` — pause agent loop, resolve in button event handler |
| `yield return` in catch | Not allowed in C# — use `string? err = null` outside try/catch |
| SSE streaming EOF | Check `ReadLineAsync() == null`, not `reader.EndOfStream` |
| File locked during publish | `Stop-Process -Name "OrchestratorIDE" -Force` before dotnet publish |
| Shared control style | Define in App.xaml only — never inline or in Window.Resources |
| WPF DataTemplate binding | Only bind to public properties on the ItemsSource collection type |
| New sidebar panel | Add as `private readonly XPanel _xPanel` in MainWindow, wire in activity bar button |

### Building Custom Security Tools in The Orc

When the user asks to build a security tool, the workflow is:
1. **Design phase** — agree on protocol/target/capability before writing code
2. **Scaffold** — `run_shell("dotnet new console -n ToolName")` or Python file
3. **Core logic** — network sockets, packet crafting, parsing, output formatting
4. **Test against lab target** — verify on controlled host before deployment
5. **Commit** — clean commit with scope of capability documented

**Common C# security tool patterns:**
```csharp
// Raw TCP connect
var client = new TcpClient();
await client.ConnectAsync(host, port);

// HTTP with custom headers (no redirects, raw control)
var handler = new HttpClientHandler { AllowAutoRedirect = false };
var http = new HttpClient(handler);
http.DefaultRequestHeaders.Add("X-Custom", "value");

// Async port scanner skeleton
var tasks = Enumerable.Range(1, 65535)
    .Select(port => Task.Run(async () => {
        using var tcp = new TcpClient();
        try { await tcp.ConnectAsync(host, port).WaitAsync(TimeSpan.FromMilliseconds(200)); return port; }
        catch { return -1; }
    }));

// Process runner (for invoking external security tools)
var psi = new ProcessStartInfo("nmap", args) { RedirectStandardOutput = true, UseShellExecute = false };
using var proc = Process.Start(psi)!;
var output = await proc.StandardOutput.ReadToEndAsync();
```

**Python security tool patterns (for OSINT app / scripts):**
```python
# HTTP with full control
import requests
r = requests.get(url, proxies={"http": "http://127.0.0.1:8080"}, verify=False, allow_redirects=False)

# Socket raw
import socket
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.settimeout(1)
result = s.connect_ex((host, port))

# Scapy (packet crafting)
from scapy.all import *
pkt = IP(dst=target)/TCP(dport=80, flags="S")
resp = sr1(pkt, timeout=1, verbose=0)
```

### Engagement Mindset Rules

```
WHEN GIVEN A TARGET:
  → Start with passive recon before active scanning
  → Map the attack surface completely before choosing entry points
  → Document every finding immediately (don't rely on memory)

WHEN A TECHNIQUE FAILS:
  → Enumerate more — you probably missed something
  → Check for non-standard ports, services, or configurations
  → Try a different protocol or angle, not just the same tool with different flags

WHEN YOU HAVE ACCESS:
  → Enumerate before you act (whoami, net user, ifconfig, ps)
  → Understand the environment fully before escalating or moving laterally
  → Leave minimal footprint — know what logs you're generating

WHEN REPORTING:
  → Lead with business impact, not technical detail
  → Every finding needs: description, reproduction steps, evidence, severity, fix
  → Severity = Likelihood × Impact (use CVSS as a guide, not gospel)
```
