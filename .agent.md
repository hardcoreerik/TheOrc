## The 7 Available Tools

| Tool | Purpose | Requires Approval |
|---|---|---|
| `read_file` | Read any file | No |
| `write_file` | Write/overwrite file | YES — DiffViewer diff shown |
| `list_files` | Directory listing | No |
| `run_shell` | PowerShell command | YES — ShellApprovalCard shown |
| `grep_code` | ripgrep search | No |
| `get_outline` | Class/method structure | No |
| `run_tests` | Run test suite | No |

**NEVER use tools that don't exist.** Common mappings:
- `create_project` → `run_shell("dotnet new winforms -n Name")`
- `install_package` → `run_shell("dotnet add package Name")`
- `design_form` / `add_control` → `write_file` with XAML content
- `build_project` → `run_shell("dotnet build")`
- `run_application` → `run_shell("dotnet run")`

## Adding a New Tool

1. Write handler in appropriate `Tools\*.cs`:
   ```csharp
   public static Task<string> MyTool(Dictionary<string, object?> args, CancellationToken ct)
   ```

2. Register in `MainWindow.RegisterAllTools()`:
   ```csharp
   _registry.Register(new ToolDefinition {
       Name        = "my_tool",
       Description = "What it does",
       Parameters  = new() { { "param_name", "string" } },
       Required    = ["param_name"],
       Handler     = MyToolsClass.MyTool
   });
   ```

3. Add to `ToolRegistry.GetForProfile()` allowed list for `Coding` or `Full`
4. Add a hint in `ToolRegistry.BuildHint()` for any common hallucinated alias names
5. Update this file with the new tool

## WPF Patterns & Gotchas

| Situation | Right approach |
|---|---|
| Equal-width tabs | `UniformGrid Rows="1"` as ItemsPanel + disabled H-scroll on wrapper |
| Async UI approval | `TaskCompletionSource<T>` — pause agent loop, resolve in button event handler |
| `yield return` in catch | Not allowed in C# — use `string? err = null` outside try/catch |
| SSE streaming EOF | Check `ReadLineAsync() == null`, not `reader.EndOfStream` |
| File locked during publish | `Stop-Process -Name "OrchestratorIDE" -Force` before dotnet publish |
| Shared control style | Define in App.xaml only — never inline or in Window.Resources |
| WPF DataTemplate binding | Only bind to public properties on the ItemsSource collection type |
| New sidebar panel | Add as `private readonly XPanel _xPanel` in MainWindow, wire in activity bar button |

## Debugging Reference

| Symptom | Root cause | Fix |
|---|---|---|
| Model says "I cannot / I'm sorry" | Refusal — no tool calls returned | AgentLoop.IsRefusal() + nudge (already in place) |
| Fake JSON blocks in plan | Model embedded ```json tool calls``` | StripFakeToolBlocks() in PlanAsync (already in place) |
| create_project not found | Model hallucinated tool | UnknownToolCard + Layer-1 hint |
| File locked on publish | OrchestratorIDE.exe is running | Stop-Process before publish |
| Tab items wrong width | StackPanel used as ItemsPanel | Switch to UniformGrid Rows="1" |
| XAML forward-reference crash | Style defined in Window.Resources | Move all shared styles to App.xaml |
| Approval never fires | OnApprovalRequested not wired | Check _approvals.ApprovalRequested += OnApprovalRequested in MainWindow ctor |
