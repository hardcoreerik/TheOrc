## Executive Summary (Top 5 Most Dangerous Gaps)

- **Data poisoning gate is absent and unaddressed post-v2**: 85% unscreened synthetic (Cerebras+Codex) with 57-83% TESTER-write misroutes poisoned ORC ACADEMY v2; prescreen_captures.py + phase3_preflight.py + finalize_training_set.py only catch mechanicals/goal-dupes on *captures*, not content-suitability for declared target "boss". No pre-train automated filter exists. (finalize_training_set.py:133 (goal_text dedup), prescreen_captures.py:126, phase3_preflight.py:178, no score_suitability.py)
- **ContentAwareDatasets_v1.md is pure vaporware**: Declares Migration v7 + example_suitability + SuitabilityRepository + VirtualDatasetService + score_suitability.py + leakage guards via example_hash; Migrations.cs stops at v6; no such tables/code exist. Virtual train_{KEY}/eval_{KEY} materialization + declared-vs-derived not implemented. The exact safeguard against v2 regression does not exist. (Migrations.cs:24, ContentAwareDatasets_v1.md:49-157, 348)
- **Rubric-in-the-loop is new, narrow, and state-unsafe**: rubric_callback.py + train_lora.py override metric only when --rubric (default on); templates still eval_loss; generation happens on live trainer model state (use_cache toggles, no empty_cache, slice gen on 16GB); resume path passes resume_from_checkpoint but callback re-inits from current trainer state without proven restoration; after_frac=0.4 + slice=24 are unvalidated heuristics. v2 summary shows the loss-vs-behavior trap is *still* possible if flag not passed. (train_lora.py:160, rubric_callback.py:58-93, qlora_job_template.json:75, training_summary.json:7 (0.2595 loss))
- **Eval integrity contract mismatch + gameability risk**: Python eval_adapter._resolve_lane + score_plan (no_tester_write etc.) mirrors SwarmSession.ParseBossPlan alias (SwarmSession.cs:2407), but C# EvalRubric.cs uses Phase-1 placeholders (ApiContract/DomainAccuracy always 10 when jsonValid); capture uses C# score while post-train A/B uses Python scorer; no enforced test that high-rubric plans survive runtime ParseBossPlan + worker lanes. A model can pass the 5 gates while emitting wrong roles or unexecutable plans. (eval_adapter.py:38, SwarmSession.cs:2407-2429, EvalRubric.cs:49, DatasetCapture.cs:63)
- **Reproducibility, OOM, and UX failure modes are unhardened**: train_lora.py sets zero seeds (torch/hf/ds); two identical --train/--eval runs can diverge; rubric gen inside on_evaluate can OOM the 16GB card with trainer state + 24-example slice held; ForgeTimer detects stale progress.json (good) but kill leaves checkpoints in unknown state and no auto-clean; GUI launch via cmd.exe shell redirect swallows detailed Python tracebacks; no pre-train validation that train/eval pair follows registry pairing and are disjoint by content hash. (train_lora.py:104 (no seed), rubric_callback.py:82 (generate during train), TrainingPitPanel.axaml.cs:835 (beatAge), 805 (Kill), finalize_training_set.py:133 (goal only))

## Top 5 Recommended Next Builds (Ordered by Leverage)

1. **Implement a minimal deterministic content-suitability pre-train gate** (Python scorer emitting per-example boss_fit / tester_fit etc. + block in phase3_preflight + review export) — directly prevents v2 regression from ever happening again; unblocks safe use of mixed corpus.
2. **Wire rubric_pass_pct as the *unconditional* default in train_lora + update templates + add golden resume + tiny-model test** — makes "loss is a proxy" a hard invariant instead of an optional flag that can be forgotten; catches behavior regressions mid-run.
3. **Add content hash (example_hash) at export time + disjoint check in preflight + finalize** — the cheapest leakage guard that works today without the full SQL virtual dataset machinery.
4. **Single source of truth for role alias map** (roles.json or shared module loaded by both C# ParseBossPlan and Python _resolve_lane/score_plan) + parity test — eliminates future drift between the capture scorer and the training/eval scorer.
5. **Harden rubric callback + Forge OOM/resume story** (isolated eval model or offload, empty_cache, per-gen timeout, structured error in progress.json, checkpoint manifest) — makes the 16 GB "rubric in the loop" safe on the stated hardware baseline instead of a source of silent corruption or wasted runs.
